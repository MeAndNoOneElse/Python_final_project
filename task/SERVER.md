# SplitChek — Полная спецификация API эндпоинтов

> **Важно:** Система поддерживает два типа расходов:
> - **Товары** — попадают на склад (холодильник/кладовка)
> - **Услуги и события** — не попадают на склад (ресторан, аренда, такси, коммуналка)

---

## Группы пользователя

### GET `/api/groups`
Возвращает список всех групп, в которых состоит текущий пользователь и бот. Для каждой группы показывает:
- название группы
- режим (family/friends)
- количество участников
- баланс пользователя в группе (сколько должен и сколько должны ему)

### POST `/api/groups`
Добавляет бота в существующий Telegram-чат. Принимает:
- ID чата
- название группы
- режим работы

### GET `/api/groups/{id}`
Возвращает детальную информацию о конкретной группе:
- все настройки группы
- список участников с их ролями
- дату создания
- статистику по расходам и долгам

### PUT `/api/groups/{id}`
Обновляет настройки группы. Можно изменить:
- название
- режим (family/friends)
- параметры уведомлений
- настройки автоматического добавления в список покупок

### DELETE `/api/groups/{id}`
Удаляет бота из группы. При этом удаляются все данные группы из базы:
- все чеки и расходы
- все холодильники и остатки
- список покупок
- долги

## Холодильники (места хранения) — только для товаров

### GET `/api/groups/{id}/storage`
Возвращает список всех холодильников/кладовок в указанной группе. Для каждого показывает:
- название
- ID
- является ли основным
- количество продуктов внутри

### POST `/api/storage`
Создаёт новый холодильник в группе. Требует:
- ID группы
- название холодильника
- флаг основного (можно только один основной на группу)

### PUT `/api/storage/{id}`
Изменяет параметры холодильника:
- переименовать
- сделать основным (при этом у старого основного снимается этот статус)

### DELETE `/api/storage/{id}`
Удаляет холодильник вместе со всеми остатками продуктов внутри. Перед удалением запрашивает подтверждение, так как операция необратима.

---

## Остатки продуктов (инвентаризация) — только для товаров

### GET `/api/storage/{id}/inventory`
Возвращает все продукты в конкретном холодильнике. Для каждого продукта показывает:
- название
- текущее количество
- единицу измерения
- минимальный порог
- прогноз дней до пустоты (на основе истории списаний за 14 дней)
- когда в последний раз обновлялся

### POST `/api/inventory/add`
Пополняет запас продукта (пользователь купил что-то). Принимает:
- ID продукта или название (если продукт новый)
- количество
- холодильник
Если продукта с таким названием нет в этом холодильнике — создаётся новая запись.

### POST `/api/inventory/consume`
Списывает продукт (съели, выбросили, закончилось). Принимает:
- ID продукта
- количество
Записывает факт списания в историю для будущих прогнозов. Если количество становится нулевым — продукт остаётся в списке с нулём, но появляется в списке покупок (если включена автоматическая генерация).

### POST `/api/inventory/move`
Перемещает продукт из одного холодильника в другой. Принимает:
- ID продукта
- из какого холодильника
- в какой холодильник
- количество
Уменьшает остаток в одном и увеличивает в другом.

### PUT `/api/inventory/{id}/threshold`
Устанавливает минимальный порог для продукта. При достижении этого порога продукт автоматически добавляется в список покупок (если включена соответствующая настройка).

## Список покупок — только для товаров

### GET `/api/groups/{id}/shopping-list`
Возвращает список покупок для всей группы. Для каждого пункта показывает:
- название продукта
- количество
- в какой холодильник нужно купить
- критичность (осталось меньше 2 дней или ниже порога)
- отмечено ли как купленное

### POST `/api/shopping-list`
Добавляет продукт в список покупок. Принимает:
- ID группы
- название продукта
- количество
- желаемый холодильник (куда потом добавить)

### PUT `/api/shopping-list/{id}`
Изменяет существующую запись в списке покупок:
- количество
- холодильник назначения

### DELETE `/api/shopping-list/{id}`
Удаляет продукт из списка покупок (например, передумали покупать).

### POST `/api/shopping-list/{id}/buy`
Отмечает продукт как купленный. Автоматически:
- добавляет продукт в указанный холодильник с указанным количеством
- удаляет запись из списка покупок

## Расходы (универсальные — и товары, и услуги)

> **Важно:** Этот раздел объединяет любые траты. Товарные чеки могут дополнительно пополнять склад, а услуги/события — нет.

### POST `/api/expenses/scan-qr`
Принимает строку из QR-кода чека (магазин, ресторан, АЗС). Парсит параметры, запрашивает данные из ФНС/ОФД. Возвращает структурированный список позиций. Пользователь сам решит:
- это товары (пополнят холодильник)
- или разовая услуга/трата (не пополняет склад)

### POST `/api/expenses/upload-photo`
Принимает фотографию чека. Запускает OCR, распознаёт товары/услуги. Возвращает результат. Аналогично — пользователь выбирает тип расхода.

### POST `/api/expenses/manual`
Создаёт расход вручную. Принимает:
- название (например, «Ужин в итальянском ресторане»)
- сумму
- категорию (продукты / ресторан / аренда / коммуналка / такси / развлечения)
- дату
- список участников, между которыми делится (если не вся группа)
- флаг `is_grocery`: true — товары, false — услуга/событие
- если `is_grocery = true` — дополнительно: список товаров и холодильник

### GET `/api/groups/{id}/expenses`
Возвращает список всех расходов в группе (и товарных, и нет). Для каждого показывает:
- дату
- кто добавил
- название
- сумму
- категорию
- тип (товары / услуга)
- статус распределения

### GET `/api/expenses/{id}`
Возвращает детали конкретного расхода:
- все позиции (если товарный чек)
- текущее распределение по участникам
- комментарий

### POST `/api/expenses/{id}/distribute`
Сохраняет распределение расхода по людям. Принимает для каждой позиции ID пользователя или NULL (общее). Для услуг/событий обычно одна позиция на всю сумму.
- **friends**: запускает расчёт долгов, создаёт запросы на подтверждение
- **family**: просто фиксирует, кто сколько заплатил, долги не создаются
- Если `is_grocery = true` — дополнительно пополняет склад в указанном холодильнике

### POST `/api/expenses/{id}/confirm`
Подтверждение распределения расхода (только friends). Когда все подтвердили — долги фиксируются.

## Долги (только режим friends)

### GET `/api/groups/{id}/debts`
Возвращает список всех активных (непогашенных) долгов в группе. Показывает:
- кто должен
- кому должен
- сумму
- за что (ссылка на расход)
- дату создания

### GET `/api/groups/{id}/balance`
Возвращает сводный баланс текущего пользователя в группе:
- сколько он всего должен другим
- сколько должны ему
- итоговое сальдо

### POST `/api/debts/{id}/settle`
Отмечает долг как погашенный. Инициирует процесс подтверждения:
- должник нажимает «я заплатил»
- кредитору приходит запрос на подтверждение получения денег
- после подтверждения кредитором долг помечается как погашенный

### GET `/api/debts/history`
Возвращает историю всех погашенных долгов пользователя по всем группам. Показывает:
- дату погашения
- сумму
- кому/от кого
- в какой группе

---

## Подтверждения и уведомления

### GET `/api/confirmations/pending`
Возвращает список всех ожидающих подтверждения операций для текущего пользователя:
- запросы на подтверждение долга
- запросы на подтверждение погашения
- запросы на подтверждение распределения расхода

### POST `/api/confirmations/{id}/approve`
Подтверждает операцию.

### POST `/api/confirmations/{id}/reject`
Отклоняет операцию.

---

## Пользователи

### GET `/api/me`
Возвращает информацию о текущем пользователе на основе Telegram initData:
- Telegram ID
- имя
- username
- настройки пользователя (язык, холодильник по умолчанию)

### PUT `/api/me`
Обновляет настройки пользователя:
- холодильник по умолчанию
- настройки уведомлений
- предпочтения по единицам измерения

---

## Вспомогательные эндпоинты

### GET `/api/groups/{id}/members`
Возвращает список участников группы с их балансами и ролями.

### GET `/api/products/search`
Поиск продуктов по части названия в рамках группы (только для товарных расходов).

### GET `/api/stats/dashboard`
Возвращает сводную статистику для главного экрана WebApp:
- количество групп
- общая сумма долгов
- количество продуктов, скоро закончатся
- количество неподтверждённых расходов
