1. Выбор группы
Сценарий: Пользователь открывает WebApp впервые.

Приложение показывает список всех групп пользователя

На карточке каждой группы: название, режим (семья/друзья), баланс

Пользователь нажимает на группу → переход на главный экран

2. Просмотр дашборда
Сценарий: Пользователь хочет увидеть общую картину по группе.

Главный экран показывает:

Мой баланс (должен / должны мне)

Тревоги (продукты на грани списания, неподтверждённые чеки)

Быстрые действия (сканер QR, добавить расход, добавить продукт)

Последние 5 расходов

Нажатие на тревогу → переход к соответствующему разделу

3. Сканирование QR-чека
Сценарий: Пользователь сканирует QR-код с чека.

Нажимает «Сканировать чек» на дашборде

Открывается камера, сканирует QR

Приложение отправляет данные на бэкенд

Бэкенд получает чек из ФНС

Автоматический переход на экран распределения расходов

4. Распределение расходов
Сценарий: Пользователь распределяет товары из чека по участникам.

На экране распределения видит список товаров

Для каждого товара выбирает:

Конкретного участника

«Общее» (делится на всех)

«Разделить пропорционально» (настраиваемые доли)

Снизу обновляется предпросмотр: кто кому сколько должен

Нажимает «Сохранить распределение»

В режиме «друзья»: участникам приходят запросы на подтверждение

В режиме «семья»: долги не создаются

5. Ручное добавление расхода
Сценарий: Пользователь добавляет расход без чека (услуга, аренда, ресторан).

Нажимает «+ Новый расход» в разделе «Расходы»

Заполняет форму:

Название (например, «Ужин в ресторане»)

Сумма

Категория

Тип (товары на склад / услуга или событие)

Дата

Нажимает «Создать»

Переход на экран распределения (одна позиция на всю сумму)

Распределяет между участниками и сохраняет

6. Просмотр холодильника
Сценарий: Пользователь смотрит, какие продукты есть в наличии.

Переходит в раздел «Холодильник»

Переключает холодильники (дом, дача, офис)

Видит таблицу продуктов:

Название

Количество

Прогресс-бар

Прогноз дней до пустоты

Критичные продукты (осталось < порога) подсвечены жёлтым

7. Списание продукта
Сценарий: Продукт закончился, нужно списать со склада.

В холодильнике находит продукт

Нажимает ⋮ → «Списать»

Указывает количество

Подтверждает → остаток уменьшается

Если остаток стал ниже порога → продукт автоматически добавляется в список покупок

8. Пополнение продукта
Сценарий: Пользователь купил продукты и добавил их в холодильник.

Вариант А — через холодильник:

В холодильнике находит продукт

Нажимает ⋮ → «Пополнить»

Указывает количество

Подтверждает → остаток увеличивается

Вариант Б — через список покупок:

Переходит в «Список покупок»

Напротив продукта нажимает «Куплено»

Подтверждает количество и холодильник

Продукт добавляется в холодильник и удаляется из списка

9. Добавление продукта в список покупок
Сценарий: Пользователь хочет запомнить, что нужно купить.

Переходит в «Список покупок»

Вводит название продукта в поле внизу

Указывает количество

Нажимает «+ Добавить»

Продукт появляется в чеклисте

Если продукт критичен (автоматически от системы) — выделяется красным

10. Просмотр долгов
Сценарий: Пользователь хочет увидеть, кому и сколько должен.

Переходит в раздел «Долги»

Видит две вкладки: «Активные долги» и «История»

В активных долгах:

Секция «Вы должны» (кому и сколько)

Секция «Должны вам» (кто и сколько)

Напротив каждого долга — кнопка «Я заплатил» (если он должник)

11. Погашение долга
Сценарий: Пользователь заплатил свой долг.

В разделе «Долги» находит свой долг

Нажимает «Я заплатил»

Подтверждает в модальном окне

Кредитору приходит запрос на подтверждение в Telegram

Кредитор подтверждает → долг исчезает из активных

Долг появляется в истории погашений

12. Перемещение продукта между холодильниками
Сценарий: Пользователь переложил продукт из дома на дачу.

В холодильнике находит продукт

Нажимает ⋮ → «Переместить»

Выбирает целевой холодильник

Указывает количество

Подтверждает → остаток уменьшается в одном и увеличивается в другом

13. Настройка порога продукта
Сценарий: Пользователь хочет, чтобы система сама напоминала, когда молока осталось мало.

В холодильнике находит продукт

Нажимает ⋮ → «Установить порог»

Вводит минимальное количество (например, 0.5 л)

Сохраняет

При достижении порога продукт автоматически попадёт в список покупок

14. Управление настройками группы
Сценарий: Пользователь (админ) меняет настройки группы.

Переходит в «Настройки»

Вкладка «О группе»:

Меняет название

Переключает режим (семья/друзья)

Копирует ссылку-приглашение

Вкладка «Холодильники»:

Добавляет новый холодильник

Удаляет пустой холодильник

Назначает основной

Вкладка «Прогнозы и пороги»:

Включает/выключает автодобавление в список покупок

Устанавливает период предупреждения (1–7 дней)

Вкладка «Уведомления»:

Выбирает, какие уведомления куда присылать

15. Подтверждение долга из Telegram (вне WebApp)
Сценарий: Пользователь получает запрос на подтверждение долга в личном чате с ботом.

Бот присылает сообщение: «Анна утверждает, что вы должны ей 500₽ за пиццу»

Пользователь нажимает кнопку «Подтвердить» (в Telegram)

Долг сохраняется в базе

При следующем открытии WebApp долг отображается в разделе «Долги»

16. Отклонение долга
Сценарий: Пользователь не согласен с долгом.

Бот присылает запрос на подтверждение

Пользователь нажимает «Отказаться»

Долг не создаётся

Инициатору приходит уведомление об отказе

Инициатор может заново распределить чек или исправить ошибку

17. Добавление нового холодильника
Сценарий: Пользователь купил новый холодильник на дачу.

Переходит в «Настройки» → вкладка «Холодильники»

Нажимает «+ Добавить холодильник»

Вводит название («Дача»)

При желании отмечает «Сделать основным»

Сохраняет

Новый холодильник появляется в переключателе на экране «Холодильник»

18. Удаление бота из группы
Сценарий: Группа больше не нужна, пользователь хочет удалить все данные.

Переходит в «Настройки» → вкладка «О приложении»

Нажимает «Удалить бота из группы»

Подтверждает в модальном окне («Все данные группы будут удалены»)

Бот выходит из чата

Все данные группы удаляются из базы

Пользователь возвращается к списку групп

19. Просмотр истории расходов
Сценарий: Пользователь хочет увидеть, сколько потратили за месяц.

Переходит в раздел «Расходы»

Видит все расходы в хронологическом порядке

Использует фильтры (по дате, по категории, по участнику)

Нажимает на любой расход → видит детали и распределение

20. Быстрое добавление продукта через чат (вне WebApp)
Сценарий: Пользователь пишет в групповой чат «купил молоко» — продукт добавляется без открытия WebApp.

Пользователь пишет в групповом чате: «купил молоко»

Бот распознаёт сообщение и уточняет: «В какой холодильник?»

Пользователь выбирает из кнопок

Бот добавляет продукт

При следующем открытии WebApp холодильник обновлён
