# SplitChek — Архитектура проекта

## Идея

Telegram-бот + веб-приложение (WebApp) для управления общими финансами и продуктами в группах. Поддерживает два режима: семейный (без долгов) и дружеский (с долгами). Позволяет отслеживать остатки в нескольких холодильниках (дом, дача, офис) и прогнозировать, когда продукты закончатся.

## Архитектура (словами)

**Три точки входа в систему:**

1. **Групповой чат Telegram** — пользователи пишут сообщения, бот их читает. Распознаёт фразы вроде «купил молоко», «молоко кончилось», «надо купить хлеб» и обновляет данные.

2. **Личный чат с ботом** — здесь бот отправляет кнопку для открытия WebApp. Также приходят уведомления о подтверждениях долгов и запросы на распределение товаров.

3. **WebApp** — полноценное веб-приложение, открывающееся внутри Telegram. Там пользователь видит дашборды, распределяет товары из чеков, управляет холодильниками и списками покупок.

**Бэкенд** — один Python процесс, внутри которого работают два сервера: FastAPI (отдаёт REST API для WebApp) и Telegram Bot (обрабатывает команды и сообщения). Они разделены по разным портам, но используют одну базу данных.

**База данных** — PostgreSQL. Хранит пользователей, группы, участников, чеки, товары, холодильники, остатки продуктов, долги и ожидающие подтверждения операции.

**Обработка чеков** — сначала пытается распарсить QR-код (если есть) и запросить полный чек из ФНС через API ОФД. Если не получается — падает на OCR через EasyOCR.

## Структура базы данных (основные таблицы)

**users** — пользователи: Telegram ID, имя, username.

**groups** — группы (Telegram чаты): ID чата, название, режим (family/friends).

**group_members** — связь пользователей с группами.

**storage_locations** — холодильники/кладовки: название, к какой группе относится, является ли основным.

**inventory** — остатки продуктов: ссылка на холодильник, название продукта, количество, единица измерения, минимальный порог.

**shopping_list** — список покупок: группа, холодильник, продукт, количество, отмечено ли купленным.

**receipts** — чеки: группа, кто загрузил, общая сумма, статус (ожидает распределения / распределён).

**receipt_items** — товары из чека: название, цена, кому назначен (если NULL — общее).

**debts** — долги (только дружеский режим): кто кому должен, сумма, погашен или нет.

**pending_confirmations** — ожидающие подтверждения: кому, какого типа (долг / распределение), данные в JSON, когда истекает.

## API эндпоинты (основные)

**GET /api/groups** — список всех групп пользователя с балансами.

**GET /api/groups/{id}/storage** — список холодильников в группе.

**GET /api/storage/{id}/inventory** — содержимое конкретного холодильника с прогнозом дней до пустоты.

**POST /api/receipts/scan-qr** — принимает строку из QR-кода чека, парсит, запрашивает данные из ФНС, возвращает структурированный список товаров.

**POST /api/receipts/{id}/distribute** — сохраняет распределение товаров по людям, пересчитывает долги, создаёт запросы на подтверждение.

**POST /api/inventory/consume** — списывает продукт (например, «молоко кончилось»).

**POST /api/inventory/move** — перемещает продукт между холодильниками.

**GET /api/groups/{id}/debts** — список активных долгов в группе.

## Frontend (React)

**Точка входа** — Telegram WebApp, который открывается по кнопке из бота. Приложение получает initData с данными пользователя и отправляет на бэкенд для верификации.

**Страницы:**

- **Группы** — список всех групп пользователя, переключение между ними.
- **Холодильник** — таблица остатков с прогресс-барами, кнопками списания и пополнения, переключение между холодильниками.
- **Список покупок** — чеклист с приоритетами (критично / обычное).
- **Чек** — после сканирования QR или загрузки фото показывает товары, позволяет распределить по людям или отметить как общее.
- **Долги** — таблица кто кому должен с кнопкой «я заплатил».

**Интеграция с Telegram** — через хук useTelegram получает доступ к WebApp API (закрытие, кнопка назад, initData).

## Обработка сообщений в чате (NLU)

Бот анализирует все сообщения в группе. Простые регулярные выражения определяют намерение:

- «купил/купила/взял X» → добавить X в холодильник (выбранный по умолчанию или через кнопки)
- «кончился/закончился/нету X» → списать X со склада
- «надо купить/добавь X» → добавить X в список покупок
- «отдал/заплатил X рублей за Y» → добавить трату, предложить разделить

Если уверенность низкая — бот переспрашивает кнопками.

## Обработка чеков (два способа)

**Способ 1 (приоритетный): QR-код**

Пользователь сканирует QR-код с чека через WebApp. Бэкенд парсит строку (t=..., s=..., fn=...), отправляет запрос в API ОФД, получает полный JSON с перечнем товаров. Данные точные на 100%.

**Способ 2 (запасной): OCR**

Если QR не читается или нет интернета — пользователь фоткает чек. Бэкенд прогоняет фото через EasyOCR, получает текст, парсит его регулярными выражениями (ищет цены и названия товаров). Возможны ошибки — пользователь может поправить вручную.
**Способ 3:
Ввод руками, если нет чека.

## Расчёт долгов (только дружеский режим)

1. После распределения товаров из чека вычисляется, сколько каждый потратил лично на себя, а сколько — на общее.
2. Общие траты делятся поровну на всех участников группы.
3. Формируется чистый баланс: кто кому сколько должен.
4. Жадный алгоритм находит минимальное количество переводов (самый крупный долг гасится самым крупным кредитором).
5. Каждому должнику приходит запрос на подтверждение в личный чат с ботом. Пока не подтвердил — долг не считается финальным.

---

## Прогноз расхода продуктов

Для каждого продукта в холодильнике бот хранит историю списаний за последние 14 дней. Вычисляется средний расход в день. Текущий остаток делится на этот расход — получаем прогнозируемое количество дней до пустоты.

Если осталось меньше 2 дней или остаток ниже минимального порога — продукт автоматически добавляется в список покупок с пометкой «критично».

## Веб-приложение (WebApp)

### Управление группами
- Список всех групп, в которых состоит пользователь и бот
- Подключение к существующей группе через ссылку-приглашение
- Переключение между группами
- удаление бота из группы

### Настройки бота
- настройка локаций (дом/дача/офис)
- Настройка минимальных порогов для продуктов (при каком остатке добавлять в список покупок)
- Включение/отключение автоматического добавления в список покупок
- Настройка периода прогноза (за сколько дней предупреждать)
- Управление уведомлениями (что присылать в чат, что — в личку)
- считать долги или нет для каждой группы

### Интерфейс для распределения расходов
- Список неподтверждённых чеков в группе
- Отображение товаров из чека (название, цена, количество)
- Выбор, кому достался каждый товар (конкретный человек / все общее / разделить пропорционально)
- Предпросмотр итогового расчёта (кто кому сколько должен)
- Кнопка подтверждения и отправки запросов участникам
- История обработанных чеков

### Дополнительные экраны
- Холодильник: остатки продуктов по местам хранения, кнопки списания/пополнения
- Список покупок: чеклист с приоритетами, отметка о покупке
- Долги: таблица кто кому должен, кнопка «я заплатил» и подтверждение, что второй заплатил

## Бэкенд (REST API)

### Группы
- `GET /groups` — список групп пользователя
- `POST /groups` — добавить бота в группу
- `GET /groups/{id}` — детали группы
- `PUT /groups/{id}` — изменить настройки бота для граппы
- `DELETE /groups/{id}` — удалить бота из группы

### Холодильники и запасы
- `GET /groups/{id}/storage` — список холодильников в группе
- `POST /storage` — создать холодильник
- `PUT /storage/{id}` — изменить холодильник
- `DELETE /storage/{id}` — удалить холодильник
- `GET /storage/{id}/inventory` — остатки продуктов
- `POST /inventory/add` — пополнить продукт (купили)
- `POST /inventory/consume` — списать продукт (съели/кончилось)
- `POST /inventory/move` — переместить продукт между холодильниками
- `PUT /inventory/{id}/threshold` — установить минимальный порог

### Список покупок
- `GET /groups/{id}/shopping-list` — список покупок группы
- `POST /shopping-list` — добавить продукт в список
- `PUT /shopping-list/{id}` — изменить количество
- `DELETE /shopping-list/{id}` — удалить из списка
- `POST /shopping-list/{id}/buy` — отметить как купленное (списывается со склада)

### Чеки и расходы
- `POST /receipts/scan-qr` — обработать QR-код чека (→ ФНС API)
- `POST /receipts/upload-photo` — загрузить фото чека (→ OCR)
- `POST /receipts/manual` — создать чек вручную (пользователь вводит товары)
- `GET /groups/{id}/receipts` — список чеков группы
- `GET /receipts/{id}` — детали чека с товарами
- `POST /receipts/{id}/distribute` — сохранить распределение товаров
- `POST /receipts/{id}/confirm` — подтвердить распределение (для участников)

### Долги (только дружеский режим)
- `GET /groups/{id}/debts` — все активные долги
- `GET /groups/{id}/balance` — баланс пользователя в группе (кому должен / кто ему)
- `POST /debts/{id}/settle` — отметить долг как погашенный
- `GET /debts/history` — история погашенных долгов

### Уведомления и подтверждения
- `GET /confirmations/pending` — список ожидающих подтверждений для пользователя
- `POST /confirmations/{id}/approve` — подтвердить
- `POST /confirmations/{id}/reject` — отклонить

### Пользователи
- `GET /me` — информация о текущем пользователе (из Telegram initData)
- `PUT /me` — настройки пользователя

## Telegram Бот (в группе)

### Добавление чека

**Способ 1 — по QR-коду**
- Пользователь пишет `/qr` или нажимает кнопку «Скан QR»
- Бот открывает WebApp со сканером QR
- После сканирования бот получает данные, запрашивает чек из ФНС
- Бот отправляет в чат сообщение: «📸 Чек на X₽ из [магазин]. Распределить? [Открыть в WebApp]»

**Способ 2 — по OCR (фото чека)**
- Пользователь отправляет фото в чат
- Бот распознаёт текст через EasyOCR
- Бот отправляет сообщение: «📸 Распознано: [список товаров]. Всё верно? [Да] [Править]»
- После подтверждения — ссылка на WebApp для распределения

**Способ 3 — вручную**
- Пользователь пишет `/add товар 100р, товар2 50р`
- Бот парсит команду, создаёт чек
- Бот отправляет ссылку на WebApp для распределения

### Другие команды в группе
- `/storage` — показать остатки в холодильнике по умолчанию
- `/shopping` — показать список покупок
- `/help` — список команд
- Естественный язык (распознавание без команд):
  - «купил молоко» → добавить в холодильник
  - «молоко кончилось» → списать со склада
  - «надо купить хлеб» → добавить в список покупок

## Telegram Бот (в личке)

### Вывод информации о группах
- `/groups` — список всех групп, где участвует пользователь
  - Для каждой группы: название, режим, количество участников
  - Кнопка «Открыть в WebApp» для каждой группы
- `/status` — сводка по всем группам (сколько должен / сколько должны ему)

### Долги (для дружеских групп)
- `/debts` — список всех активных долгов по всем группам
  - Где и кому должен: «В группе [Коттедж] → Коле 500₽»
  - Где должны ему: «В группе [Коллеги] → Маша должна 230₽»
- `/debts [группа]` — долги по конкретной группе
- `/pay [группа] [кому] [сумма]` — отметить, что заплатил (отправляет подтверждение другой стороне)

### Подтверждения
- Бот присылает сообщение: «Коля утверждает, что вы должны ему 500₽ за пиццу. [Подтвердить] [Отказаться]»
- После подтверждения долг сохраняется в БД
- аналогично с погашением долга
### Настройки
- `/settings` — открыть WebApp с настройками (группы, пороги, уведомления)
- `/default_storage [название]` — установить холодильник по умолчанию для быстрого добавления продуктов

### Быстрое добавление
- `/take молоко 1л` — добавить продукт в холодильник по умолчанию
- `/use молоко 0.5л` — списать продукт
- `/want хлеб` — добавить в список покупок
## Ключевые особенности проекта

- Чат как основной интерфейс — не нужно открывать приложение для простых действий
- Два режима работы — семья (без долгов) и друзья (с долгами)
- Мульти-холодильники — отдельные запасы для дома, дачи, офиса
- QR-код чеков — 100% точность данных из ФНС
- Подтверждение долгов — нельзя навязать другому человеку
- Прогнозы — бот сам напоминает, когда пора покупать
