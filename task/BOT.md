1. Общая концепция
Бот работает в двух режимах:

В групповых чатах — ограниченный функционал: добавление чеков, распределение трат, работа со списком покупок

В личных сообщениях — полный доступ: ссылка на WebApp, просмотр долгов по всем группам, настройки, подтверждения

2. Функции в групповом чате
2.1. Добавление чеков
Способ	Как работает
QR-код	Пользователь пишет /qr → бот открывает WebApp со сканером → после сканирования бот публикует в чат: «📸 Чек из [магазин] на X₽. [Открыть в WebApp для распределения]»
Фото чека	Пользователь отправляет фото в чат → бот распознаёт через OCR → публикует: «📸 Распознано: [список товаров]. Всё верно? [Да] [Править]» → после подтверждения — ссылка на WebApp
Вручную	Пользователь пишет /add молоко 89₽, хлеб 45₽ → бот создаёт чек → публикует ссылку на WebApp для распределения
2.2. Распределение трат
Действие	Описание
Ссылка на WebApp	После добавления чека бот публикует кнопку «Открыть в WebApp», где пользователь распределяет товары по участникам
Уведомление о завершении	Когда кто-то распределил чек, бот пишет в чат: «✅ Чек из [магазин] распределён. [Открыть]»
2.3. Список покупок
Команда	Действие
/shopping	Показать текущий список покупок группы (в виде сообщения)
надо купить хлеб (NLU)	Распознаёт фразу и добавляет «хлеб» в список покупок, подтверждает в чате
/add_shopping хлеб 2 шт	Добавляет продукт в список покупок
2.4. Холодильник (только просмотр)
Команда	Действие
/storage	Показать остатки в основном холодильнике (кратко: продукт — количество — прогноз)
/storage дача	Показать остатки в указанном холодильнике
2.5. NLU (распознавание естественного языка)
Бот анализирует все сообщения в группе:

Фраза	Действие
«купил молоко», «купила хлеб», «взял сыр»	Добавляет продукт в холодильник (спрашивает какой холодильник, если не указан)
«молоко кончилось», «закончился хлеб», «нету сыра»	Списывает продукт со склада
«надо купить хлеб», «добавь молоко»	Добавляет продукт в список покупок
«отдал 500₽ за пиццу»	Создаёт расход, предлагает распределить через WebApp
При низкой уверенности — бот переспрашивает кнопками: «Это покупка, списание или список покупок?»

2.6. Что НЕ ДОЛЖЕН делать бот в группе
❌ Не показывать долги (кто кому должен)

❌ Не показывать личные балансы

❌ Не принимать команды /debts, /pay, /settings

❌ Не отправлять в группу запросы на подтверждение долгов

3. Функции в личном чате с ботом
3.1. Ссылка на WebApp (главный вход)
Команда	Действие
/start	Приветствие + кнопка «🚀 Открыть SplitChek» (запускает WebApp)
/webapp	Кнопка «Открыть приложение»
WebApp открывается с авторизацией через Telegram initData.

3.2. Просмотр долгов по всем группам
Команда	Действие
/debts	Показать все активные долги по всем группам:
• «В группе [Соседи] → вы должны Анне 500₽»
• «В группе [Коллеги] → Иван должен вам 230₽»
/debts [группа]	Показать долги по конкретной группе, например /debts Соседи
/debts pending	Показать долги, ожидающие вашего подтверждения
3.3. Баланс по группам
Команда	Действие
/status	Сводка по всем группам:
• [Соседи]: должны 500₽, вам должны 120₽
• [Коллеги]: должны 0₽, вам должны 230₽
/balance [группа]	Детальный баланс в конкретной группе
3.4. Погашение долгов (быстрое действие)
Команда	Действие
/pay [группа] [кому] [сумма]	Пример: /pay Соседи Анна 500 → отправляет Анне запрос на подтверждение получения денег
/pay [id долга]	Погасить долг по его ID (из списка /debts)
3.5. Подтверждения (входящие)
Бот присылает в личку запросы на подтверждение:

Тип	Сообщение	Кнопки
Новый долг	«📝 Анна утверждает, что вы должны ей 500₽ за пиццу в группе [Соседи]»	✅ Подтвердить / ❌ Отказаться
Погашение долга	«💰 Сергей заплатил вам 300₽. Подтверждаете получение?»	✅ Да, получил / ❌ Нет
Распределение чека	«🧾 Распределение чека из Пятёрочки. Ваша доля: 245₽. Согласны?»	✅ Да / ✏️ Изменить
После нажатия кнопки — бот отправляет подтверждение в систему и уведомляет инициатора.

3.6. Настройки
Команда	Действие
/settings	Открыть WebApp на странице настроек пользователя
/default_storage [название]	Установить холодильник по умолчанию для быстрого добавления продуктов через групповой чат
/notifications	Настроить, какие уведомления присылать (в личку / не присылать)
3.7. Быстрое добавление (без открытия WebApp)
Команда	Действие
/take молоко 1л	Добавить продукт в холодильник по умолчанию (выбранный через /default_storage)
/use молоко 0.5л	Списать продукт из холодильника по умолчанию
/want хлеб	Добавить продукт в список покупок группы по умолчанию
3.8. Информационные команды
Команда	Действие
/groups	Список всех групп, где участвует пользователь, с кнопкой «Открыть» для каждой
/help	Список всех команд с кратким описанием
/about	Информация о боте, версия, ссылка на инструкцию
4. Общие требования
4.1. Клавиатуры
Место	Тип	Кнопки
Личный чат, /start	Inline	🚀 Открыть SplitChek, 📊 Мои долги, ⚙️ Настройки
Личный чат, /debts	Inline	Подтвердить / Отказаться (для каждого долга)
Групповой чат, после распознавания	Inline	🛒 Покупка / 🗑️ Списание / 📝 Список покупок
Групповой чат, после добавления чека	Inline	🔗 Открыть в WebApp
4.2. Форматирование сообщений
Использовать жирный для сумм и названий

Эмодзи для визуального разделения:

🧾 — чек

💰 — долг

🧊 — холодильник

📝 — список покупок

✅ — успех

❌ — ошибка

⚠️ — предупреждение

4.3. Ограничения и безопасность
Бот не отвечает на сообщения в группе, если они не содержат команд или ключевых фраз (NLU)

Бот не читает сообщения, если не добавлен в группу как участник

Все подтверждения долгов требуют явного действия от обеих сторон

WebApp передаёт initData для авторизации, бот его не генерирует
